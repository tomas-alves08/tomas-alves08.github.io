What is Pseudo Code?

Pseudo Code is a "technique" used as a bridge in-between the algorithm and the coding itself. Algorithm is an organized logical sequence of the steps to be followed usually represented graphically.

Pseudo Code is performed by done all the steps the code will do but with human language which will later be translated into a programming language. It helps because it makes easier on the person to note down the steps they are intending to follow to code. It can also be implemented by simply breaking down the bigger tasks into smaller ones using comment lines.

How confident I feel about it..

I actually use this tool a lot. Back in the days I was working with Continues Improvement there was a planning tool that has the same idea as Pseudo Code that is pretty much to get a major task and break it down to a one-person-task-size. Breaking tasks down gives me a better way to track my daily goals. I know what I want to achieve for that day so it's easier to track my achievements and also where I am behind.

What is Code Readability?

Can you picture yourself coming into a new project and having to review, adjust or implement new features into a webpabge. However, the situation you face seems more like a war field. You cannot understand what functions are doing what, you cannot understand the conditionals applied etc. etc...

Shit Yuck GIFfrom Shit GIFs

Where do you start from? It's a hard question. It's estimates that time spent reading vs time spent coding when someone new needs to review a website is well over 10 : 1. But, how can we improve that?

There are some best practices when the topic is about code readability. The most common ones are:

  • Reuse bits of codes when you possibly can, in other words, DRY - Dont Repeat Yourself!!
  • Use identation to be able to easily identify the different blocks of codes.
  • Use self explanatory and meaninful names for variables and functions
  • Use comments to explain beginnings of different parts and other parts of codes that are not obvious.

You probably want to make the life of the next person who will have to work on your code easier. It actually can be you doing that, reading your own code. And no doubt you will work on someone else's code as well. We should all keep it tidy and clean.

How confident do I feel about it..

I am aware of the benefits of following the identation, DRY and other best practices regarding readability, however it's something that I have been practicing and getting used to. At the moment, as I am starting my journey I am more focus on getting things working but hopefully soon I will reach the level where I will be confident on programming and I will switch onto the making code simple, readable and efficient.

What is the Rubber Ducky Method

The Rubber Ducky Method is a way of making someone's mind to look at the same problem but through a different point of view.

This technique requires that a person having a problem explains what him/her is doing to sort the issue by reading the code and explaining what every single line of code means. The idea is that the person trying to explain what him/her is trying to do on each line eventually finds out what in the code is wrong.

How confident do I feel about it

I have just heard of this tichnique so I haven't explored it too much. But it's an easy and powerful technique in my opinion. The idea behind it is great.

Console Log:

In my personal opinion, console logging is one of the most powerful tools a developer has. And it is because consologging allows the programer to have an instantaneous picture of how his code looks like.

It's a good habit to console log a piece of code while you are developing it so you can see if the result of your code is what you were expecting instead of finding that out a few steps down the track and having to get back and try to look for sometimes a tiny error.
